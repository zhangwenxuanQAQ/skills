## 核心职责
- 在项目根目录下生成README.md文件
- 当有新功能模块添加或者已有功能发生重要更新时需要更新README.md
  
## 需要满足如下要求
1. 需要包含以下几个部分（可自行扩展）：
   - 项目介绍
   - 前后端技术栈
   - 代码目录结构
   - 功能模块简介
   - 配置文件以及配置项介绍
   - 依赖安装命令
   - 项目启动方法
   - 服务地址（比如swagger地址，前后端地址等）
   - 使用Docker部署指南

2. 尽量不要出现敏感信息，比如配置项中数据库连接信息，核心业务逻辑等
